Reason for choosing python:

The project demanded a lot of components to be designed, coded and tested. 
Given the time limitation, and my target to build all components in the best possible way, I chose python for its ease in implementation. I wanted to express my knowledge about multiple components, how they are designed and portray their working as a one single machine.
The code structure and flow is completely similar to what it would have been in C++ or Java.
I have coded it in a similar way to C++, have not used the third party python libraries and extensively used the concepts of OOPs - classes, objects, function delegates, inheritance, etc.
I have also tried to make sure of private and public variables at most places in all of my modules.

If given more time:

I will try to implement this complete infrastucture in C++ with all relevant features and best coding practices.
Although the test signal "HiLo" which I designed, was giving profit in this strategy, I would have designed a more sophisticated signal if given more time.
More generalization of the infrastructure to further support multiple function delegates, multiple components of all kinds could have also been implemented if I was given more time
